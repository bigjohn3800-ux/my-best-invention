## 1. 관리자 역할 개요

플랫폼은 4가지 역할을 지원합니다:

| 역할 | 코드명 | 권한 범위 |
|------|--------|----------|
| **슈퍼관리자** | superadmin | 모든 관리 기능 + 역할 변경 권한 |
| **관리자** | admin | 회원 관리, 통계, 프랜차이즈 관리 |
| **단체 관리자** | group_admin | 소속 단체 관리, 회원 진도 관리 |
| **일반 회원** | member | 학습 및 AI 도구 사용 |

### 역할별 접근 가능 메뉴

| 메뉴 | 경로 | 슈퍼관리자 | 관리자 | 단체 관리자 |
|------|------|:---:|:---:|:---:|
| 관리자 패널 | `/admin` | O | X | X |
| 회원 관리 | `/admin/members` | O | O | X |
| 단체 관리 | `/admin/group` | O | O | O |

## 2. 관리자 계정 접속

### 기본 계정 정보

| 역할 | 아이디 | 비밀번호 |
|------|--------|----------|
| 슈퍼관리자 | only-jung-dw@hanmail.net | onlyjdw3831 |
| 관리자 | inventor | 1234 |

### 접속 방법

1. `/auth` 페이지에서 해당 계정으로 로그인합니다.
2. 로그인 후 네비게이션 바에 **"관리"** 메뉴가 표시됩니다.
3. "관리" 메뉴를 클릭하면 관리자 페이지로 이동합니다.

## 3. 슈퍼관리자 기능

**경로**: `/admin`

슈퍼관리자는 플랫폼의 최고 권한을 가진 관리자입니다.

### 3.1 시스템 대시보드

관리자 패널 상단에 전체 시스템 현황이 표시됩니다:

- **총 회원수**: 가입된 전체 회원 수
- **이번 달 신규 가입**: 당월 신규 가입자 수
- **등록 단체 수**: 생성된 조직(단체) 수

### 3.2 사용자 역할 관리 (슈퍼관리자 전용)

전체 사용자 목록에서 각 회원의 역할을 변경할 수 있습니다.

#### 역할 변경 방법

1. `/admin` 페이지에서 사용자 목록을 확인합니다.
2. 변경할 사용자의 행에서 **역할 드롭다운**을 클릭합니다.
3. 변경할 역할을 선택합니다:
   - **member**: 일반 회원
   - **group_admin**: 단체 관리자 (교사/멘토)
   - **admin**: 관리자
4. 변경 사항이 즉시 적용됩니다.

#### 주의사항
- 슈퍼관리자의 역할은 변경할 수 없습니다.
- 다른 슈퍼관리자의 역할도 변경할 수 없습니다.
- 역할 변경 시 해당 사용자의 접근 권한이 즉시 변경됩니다.

## 4. 일반 관리자 기능

### 4.1 회원 관리

**경로**: `/admin/members`

#### 회원 검색 및 조회

1. 검색창에 사용자명 또는 이름을 입력하여 검색합니다.
2. 전체 회원 목록에서 아이디, 이름, 역할, 가입일을 확인할 수 있습니다.

#### 회원 차단/차단 해제

문제가 있는 회원을 차단할 수 있습니다.

1. 회원 목록에서 해당 사용자를 찾습니다.
2. **"차단"** 버튼을 클릭합니다.
3. 차단된 사용자는 로그인이 불가능해집니다.
4. 차단을 해제하려면 **"차단 해제"** 버튼을 클릭합니다.

#### 차단 제한사항
- 관리자(admin)는 다른 관리자나 슈퍼관리자를 차단할 수 없습니다.
- 슈퍼관리자만 관리자를 차단할 수 있습니다.

## 5. 단체 관리자(교사/멘토) 기능

**경로**: `/admin/group`

교사, 멘토, 학원 관리자를 위한 기능입니다.

### 5.1 단체(조직) 만들기

1. `/admin/group` 페이지에서 **"단체 만들기"** 버튼을 클릭합니다.
2. 다음 정보를 입력합니다:
   - **단체명**: 학교명, 학원명, 클래스명 등
   - **설명**: 단체에 대한 간단한 설명
3. **"만들기"** 버튼을 클릭하면 단체가 생성됩니다.

### 5.2 초대 코드 관리

단체를 생성하면 자동으로 **8자리 초대 코드**가 발급됩니다.

#### 초대 코드 사용법

1. 단체 카드에서 **초대 코드**를 확인합니다.
2. **"복사"** 아이콘을 클릭하여 클립보드에 복사합니다.
3. 학생들에게 초대 코드를 안내합니다.
4. 학생들은 회원가입 시 초대 코드를 입력하여 해당 단체에 가입합니다.

### 5.3 소속 회원 확인

단체를 선택하면 소속된 회원 목록을 확인할 수 있습니다:
- 회원 이름
- 가입 일자
- 학습 레벨

## 6. 프랜차이즈 문의 관리

**경로**: `/admin` (관리자/슈퍼관리자)

### 문의 목록 확인

프랜차이즈 페이지(`/franchise`)를 통해 접수된 문의를 관리합니다.

### 문의 정보

| 항목 | 설명 |
|------|------|
| 기관명 | 문의한 기관/학교 이름 |
| 담당자 | 문의 담당자 이름 |
| 이메일 | 연락 이메일 |
| 전화번호 | 연락 전화번호 |
| 예상 인원 | 도입 예상 인원 수 |
| 문의 내용 | 상세 문의 내용 |

### 문의 상태 관리

각 문의의 상태를 업데이트할 수 있습니다:

| 상태 | 의미 | 색상 |
|------|------|------|
| **pending** | 접수됨 (미처리) | 노란색 |
| **contacted** | 연락 완료 | 파란색 |
| **completed** | 처리 완료 | 초록색 |
| **rejected** | 거절 | 빨간색 |

#### 상태 변경 방법

1. 프랜차이즈 문의 목록에서 해당 문의를 찾습니다.
2. **상태 드롭다운**에서 새로운 상태를 선택합니다.
3. 변경 사항이 즉시 저장됩니다.

## 7. 단체(조직) 관리

### 여러 단체 관리

- 관리자와 슈퍼관리자는 **모든 단체**를 조회할 수 있습니다.
- 단체 관리자는 **본인이 만든 단체**만 조회할 수 있습니다.

### 단체 목록 정보

각 단체 카드에는 다음 정보가 표시됩니다:
- 단체명
- 설명
- 초대 코드
- 소속 회원 수

### 단체 선택

단체를 클릭하면 해당 단체의 **상세 대시보드**가 표시됩니다.

## 8. 교사 대시보드 (상세 통계)

단체를 선택하면 **"단체 현황 대시보드"**가 나타납니다.

### 8.1 요약 통계 카드 (6개)

| 카드 | 아이콘 | 설명 |
|------|--------|------|
| **회원수** | 사람 아이콘 (시안) | 단체 소속 전체 회원 수 |
| **발명 프로젝트** | 전구 아이콘 (노랑) | 소속 회원들의 총 발명 프로젝트 수 |
| **BMC 캔버스** | 로켓 아이콘 (보라) | 소속 회원들의 총 BMC 프로젝트 수 |
| **진단 횟수** | 뇌 아이콘 (초록) | 소속 회원들의 총 진단 완료 횟수 |
| **AI 사용** | 트렌드 아이콘 (파랑) | 소속 회원들의 총 AI 도구 사용 횟수 |
| **수료 과정** | 배지 아이콘 (주황) | 소속 회원들의 총 코스 수료 수 |

### 8.2 회원별 활동 현황 테이블

각 소속 회원의 상세 활동을 테이블로 확인합니다:

| 열 | 설명 |
|----|------|
| **회원** | 회원 이름 (아바타 + 이름) |
| **레벨** | 현재 학습 레벨 (초급/중급/고급) |
| **발명** | 발명 프로젝트 수 (노란색) |
| **BMC** | BMC 프로젝트 수 (보라색) |
| **진단** | 진단 완료 횟수 (초록색) |
| **AI** | AI 도구 사용 횟수 (파란색) |
| **배지** | 획득 배지 수 (주황색) |
| **수료** | 수료/진행 중 코스 수 (초록/노랑) |

### 8.3 활용 방법

#### 학습 진도 모니터링
- 각 학생의 발명/BMC 프로젝트 수를 확인하여 과제 진행 상태를 파악합니다.
- 수료 열에서 코스 완료 현황을 확인합니다.

#### AI 활용도 분석
- AI 사용 횟수를 확인하여 학생들의 AI 도구 활용도를 파악합니다.
- AI 사용이 적은 학생에게 활용을 권장할 수 있습니다.

#### 진단 결과 추적
- 진단 횟수를 확인하여 학생들의 자기평가 참여도를 파악합니다.

#### 배지 현황
- 배지 수로 학생들의 전반적인 활동 참여도를 판단합니다.

## 9. 회원 관리

### 9.1 전체 회원 조회

**경로**: `/admin/members`

관리자는 전체 회원 목록을 조회할 수 있습니다:
- 사용자 ID
- 아이디 (username)
- 표시 이름
- 역할
- 가입일

### 9.2 회원 상태 관리

#### 차단하기
1. 문제 있는 회원의 **"차단"** 버튼을 클릭합니다.
2. 차단된 회원은 즉시 로그인이 불가능해집니다.
3. 차단 상태는 회원 목록에 표시됩니다.

#### 차단 해제하기
1. 차단된 회원의 **"차단 해제"** 버튼을 클릭합니다.
2. 해당 회원이 다시 로그인할 수 있게 됩니다.

## 10. 시스템 통계

### 10.1 관리자 대시보드 통계

**경로**: `/admin`

| 통계 항목 | 설명 |
|----------|------|
| 총 회원수 | 플랫폼에 가입된 전체 회원 수 |
| 이번 달 신규 | 해당 월에 가입한 신규 회원 수 |
| 등록 단체 수 | 생성된 조직(단체) 수 |

### 10.2 공개 플랫폼 통계

홈페이지에 표시되는 공개 통계 (API: `GET /api/stats/public`):

| 항목 | 설명 |
|------|------|
| 누적 회원수 | 전체 가입 회원 수 |
| 프로젝트 | 발명 + BMC 프로젝트 총 수 |
| AI 분석 | AI 아이디어 분석 + 진단 분석 총 수 |

### 10.3 단체별 상세 통계

API: `GET /api/organizations/:id/stats`

각 단체의 회원별 상세 활동 데이터를 제공합니다:
- 발명 프로젝트 수
- BMC 캔버스 수
- 진단 횟수
- AI 사용 횟수
- 배지 수
- 수료 현황

## 11. 관리 운영 가이드

### 11.1 신규 학교/기관 온보딩 절차

1. **문의 접수**: 프랜차이즈 페이지를 통해 문의를 받습니다.
2. **문의 처리**: 관리자 패널에서 문의 상태를 "contacted"로 변경합니다.
3. **계정 준비**: 교사 계정의 역할을 "group_admin"으로 변경합니다.
4. **단체 생성**: 교사가 직접 단체를 생성하거나 관리자가 대신 생성합니다.
5. **초대 코드 전달**: 생성된 초대 코드를 교사에게 전달합니다.
6. **학생 가입**: 학생들이 초대 코드로 회원가입합니다.
7. **문의 완료**: 문의 상태를 "completed"로 변경합니다.

### 11.2 일상 관리 체크리스트

| 항목 | 빈도 | 설명 |
|------|------|------|
| 신규 문의 확인 | 매일 | 프랜차이즈 문의 접수 여부 확인 |
| 회원 현황 점검 | 주 1회 | 신규 가입자 및 활동 현황 확인 |
| 문제 회원 관리 | 수시 | 부적절한 활동 회원 차단 처리 |
| 단체 지원 | 수시 | 교사/멘토 문의 대응 |

### 11.3 문제 해결 가이드

#### 학생이 초대 코드로 가입이 안 됩니다
- 초대 코드가 정확한지 확인합니다.
- 해당 단체가 정상적으로 생성되어 있는지 확인합니다.
- 학생에게 코드를 다시 안내합니다.

#### 교사가 관리 메뉴를 볼 수 없습니다
- 해당 교사 계정의 역할이 "group_admin" 이상인지 확인합니다.
- 슈퍼관리자가 `/admin` 페이지에서 역할을 변경해야 합니다.

#### 회원이 로그인할 수 없습니다
- 해당 회원이 차단 상태인지 확인합니다.
- 차단 상태라면 차단을 해제합니다.
- 비밀번호 문제라면 관리자가 확인 후 안내합니다.

### 11.4 API 엔드포인트 참조

관리자 전용 API 목록:

| 경로 | 메서드 | 권한 | 설명 |
|------|--------|------|------|
| `/api/admin/users` | GET | 슈퍼관리자, 관리자 | 전체 사용자 목록 |
| `/api/admin/users/:id/role` | PATCH | 슈퍼관리자 | 사용자 역할 변경 |
| `/api/admin/users/:id/block` | PATCH | 슈퍼관리자, 관리자 | 사용자 차단/해제 |
| `/api/admin/stats` | GET | 슈퍼관리자, 관리자 | 시스템 통계 |
| `/api/admin/franchise` | GET | 슈퍼관리자, 관리자 | 프랜차이즈 문의 목록 |
| `/api/admin/franchise/:id/status` | PATCH | 슈퍼관리자, 관리자 | 문의 상태 변경 |
| `/api/organizations` | GET | 모든 관리자 | 단체 목록 |
